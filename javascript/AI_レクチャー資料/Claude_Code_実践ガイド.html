第2章: セットアップと基本操作

MCP（Model Context Protocol）

外部ツールやサービスと連携するためのオープンプロトコル

MCPでできること

  • GitHub連携（PR作成・レビュー）
  • データベース操作
  • Slack/Notion連携
  • ブラウザ操作
  • 独自ツールの追加

設定方法

// .mcp.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y",
        "@modelcontextprotocol/
        server-github"]
    }
  }
}

Tip MCPサーバーを追加することで、Claudeの機能を拡張できます

第2章: セットアップと基本操作

ローカルLLMでの動作

インターネット接続なしでも動作可能（プライバシー重視の環境向け）

設定方法

# Ollamaを使用する場合
export ANTHROPIC_BASE_URL=http://localhost:11434/v1
export ANTHROPIC_API_KEY=ollama

# LM Studioを使用する場合
export ANTHROPIC_BASE_URL=http://localhost:1234/v1
export ANTHROPIC_API_KEY=lm-studio

# その後、通常通り起動
claude

メリット

  • 完全オフライン動作
  • データが外部に送信されない
  • API料金が不要

注意点

  • モデル性能はローカルLLMに依存
  • 高性能GPUが必要な場合あり
  • 一部機能が制限される場合あり

第3章: 効果的なプロンプト技法

良いプロンプトとは？

良いプロンプトの特徴

  • 具体的: 何をしたいか明確
  • 文脈あり: 背景情報を含む
  • 制約明示: 条件や制限を伝える
  • 期待値明示: 出力形式を指定

悪いプロンプトの特徴

  • 曖昧: 何をしたいか不明
  • 情報不足: 必要な情報がない
  • 一度に多すぎ: 複数タスク混在
  • 前提の欠如: 共有知識を仮定

第3章: 効果的なプロンプト技法

悪いプロンプト例

バグ直して

問題点: どのバグ？どこのファイル？どんな症状？

いい感じにして

問題点: 何が「いい」の基準？どこを改善？

全部書いて

問題点: 何を？どんな仕様で？どの言語で？

第3章: 効果的なプロンプト技法

コンテキストの与え方

5W1Hを意識する

  • What: 何をしたいのか
  • Where: どのファイル・どの箇所か
  • Why: なぜその変更が必要か
  • How: どのような方法で（制約）
  • When: いつ実行されるか（条件）
【例】
What: ログイン機能を追加したい
Where: src/pages/Login.tsx（新規作成）
Why: ユーザー認証を実装するため
How: Firebase Authを使用
When: ユーザーがログインボタンを押した時

注意点とセキュリティ

注意すべきこと

  • 生成コードは必ずレビュー

  • 本番環境では慎重に
  • バックアップを取る
  • 100%正確ではない

推奨プラクティス

  • Gitで変更を追跡
  • テストで動作確認
  • 小さい変更を積み重ね
  • 不明点は質問する
  • 定期的にレビュー
