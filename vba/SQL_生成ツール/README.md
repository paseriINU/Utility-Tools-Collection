# Oracle SELECT文生成ツール

Excel VBAを使用して、Oracle用のSELECT文を対話的に生成するツールです。SQL初心者でも複雑なクエリを簡単に作成できます。

## 機能

### 基本機能
- **SELECT句**: 取得するカラムの指定、別名(AS)の設定
- **FROM句**: メインテーブルの指定
- **WHERE句**: 抽出条件の設定（AND/OR、括弧による優先順位）
- **ORDER BY句**: 並び順の指定（昇順/降順、NULLS位置）
- **DISTINCT**: 重複行の除外

### 結合（JOIN）
- INNER JOIN（内部結合）
- LEFT JOIN（左外部結合）
- RIGHT JOIN（右外部結合）
- FULL OUTER JOIN（完全外部結合）
- CROSS JOIN（交差結合）

### 集計機能
- **集計関数**: COUNT, SUM, AVG, MAX, MIN, COUNT(DISTINCT)
- **GROUP BY句**: グループ化
- **HAVING句**: 集計結果の絞り込み

### 高度な機能
- **サブクエリ**: SELECT句やWHERE句内でのサブクエリ使用
- **WITH句**: 共通テーブル式（CTE）の定義
- **UNION / UNION ALL**: 複数SELECT結果の結合
- **件数制限**: FETCH FIRST / ROWNUM方式

### 学習支援機能
- **SQLヘルプシート**: 各SQL構文の説明と使用例
- **各セクションの説明**: メインシートに各項目の簡単な説明を表示
- **サンプルデータ**: テーブル定義シートにサンプルテーブル・カラムを登録済み

### テーブル定義連動機能
- **プルダウン更新**: テーブル定義に登録したテーブル名・カラム名をプルダウンで選択可能
- **別名更新**: 入力したテーブル別名を各セクションのプルダウンに反映
- **入力ミス防止**: プルダウンから選択することでタイプミスを防止

### テーブル定義書インポート機能
- **外部ファイルからインポート**: 既存のテーブル定義書（Excelファイル）からテーブル・カラム情報を一括インポート
- **フォルダ一括処理**: 指定フォルダ内の全Excelファイルを自動処理
- **設定カスタマイズ**: テーブル名やカラム定義の位置を定義書フォーマットに合わせて変更可能
- **常に最新化**: インポート実行で既存データをクリアし、最新の定義に置換

## 必要な環境

- Microsoft Excel 2010以降
- マクロを有効にする必要があります

## インストール方法

### 方法1: VBAエディタでインポート（推奨）
1. Excelを開き、`Alt + F11`でVBAエディタを起動
2. 「ファイル」→「ファイルのインポート」を選択
3. **`SQL_生成ツール_Setup.bas`ファイルを選択してインポート**（初期化・Setup用）
4. 「ファイル」→「ファイルのインポート」を選択
5. **`SQL_生成ツール.bas`ファイルを選択してインポート**（ビジネスロジック用）
6. VBAエディタを閉じる
7. ファイルを`.xlsm`（マクロ有効ブック）として保存

**重要**: 両方のモジュールをインポートする必要があります。片方だけではエラーになります。

### 方法2: 新規ブックにコードを貼り付け
1. 新規Excelブックを作成
2. `Alt + F11`でVBAエディタを起動
3. 「挿入」→「標準モジュール」を選択
4. `SQL_生成ツール_Setup.bas`の内容を全てコピー＆ペースト
5. 「挿入」→「標準モジュール」を選択（もう一つモジュールを追加）
6. `SQL_生成ツール.bas`の内容を全てコピー＆ペースト
7. ファイルを`.xlsm`として保存

## 使い方

### 初期設定
1. Excelで`Alt + F8`を押してマクロダイアログを開く
2. `InitializeSQL生成ツール`を選択して「実行」
3. 必要なシートが自動的に作成されます

### シート構成

| シート名 | 説明 |
|---------|------|
| メイン | SQL生成の入力画面。条件を入力してSQLを生成 |
| テーブル定義 | 使用するテーブルとカラムの一覧を登録 |
| 生成履歴 | 過去に生成したSQLを保存 |
| サブクエリ | サブクエリの定義 |
| WITH句 | 共通テーブル式（CTE）の定義 |
| UNION | UNION結合するSQLの定義 |
| SQLヘルプ | SQL構文の説明と使用例 |

### 基本的な使い方

#### 1. テーブル定義の登録（推奨）
「テーブル定義」シートに、使用するテーブルとカラムの情報を登録します。

**テーブル一覧（A〜C列）に登録:**
| No | テーブル名 | 説明 |
|----|-----------|------|
| 1 | USERS | ユーザーマスタ |
| 2 | ORDERS | 注文テーブル |

**カラム一覧（E〜H列）に登録:**
| テーブル名 | カラム名 | データ型 | 説明 |
|-----------|---------|---------|------|
| USERS | USER_ID | NUMBER | ユーザーID |
| USERS | USER_NAME | VARCHAR2(100) | ユーザー名 |

登録後、メインシートの「**プルダウン更新**」ボタンをクリックすると、テーブル名やカラム名がプルダウンで選択できるようになります。

#### 1-2. テーブル定義書からのインポート（オプション）
既存のテーブル定義書（Excelファイル）がある場合は、「テーブル定義」シートの「**定義書インポート**」ボタンから一括インポートできます。

**インポート設定（テーブル定義シートのJ列で変更可能）:**
| 設定項目 | デフォルト値 | 説明 |
|---------|-------------|------|
| テーブル名セル | E4 | 各定義書でテーブル名が記載されているセル |
| カラム開始行 | 10 | カラム定義が開始する行番号 |
| カラム番号列 | A | カラム番号が記載されている列 |
| カラム名列 | B | カラム名が記載されている列 |
| 説明列 | C | カラム説明が記載されている列 |
| データ型列 | D | データ型が記載されている列 |
| フォルダ1 | (空) | 定義書フォルダパス1（設定するとダイアログ省略可） |
| フォルダ2 | (空) | 定義書フォルダパス2（複数フォルダ対応） |

**インポート手順:**
1. 「テーブル定義」シートで設定を確認（必要に応じて変更）
2. 「**定義書インポート**」ボタンをクリック
3. フォルダ選択方法を選択:
   - **フォルダパスを事前設定している場合**: 設定済みフォルダを使用するか選択
   - **手動選択の場合**: 1つまたは2つのフォルダを選択
4. 選択したフォルダ内のExcelファイルから自動的にテーブル・カラム情報をインポート（既存データは自動的にクリアされます）

**複数フォルダからのインポート:**
テーブル定義書が複数の場所に分散している場合、2つのフォルダから一括インポートできます：
- K11セルにフォルダ1のパスを設定
- K12セルにフォルダ2のパスを設定
- インポート実行時に設定済みフォルダを使用するか選択可能

#### 2. メインシートでの入力

**メインテーブルの指定**
```
テーブル名: USERS
別名: u
```

**結合テーブルの指定（必要な場合）**
```
結合種別: INNER JOIN
テーブル名: ORDERS
別名: o
結合条件: u.USER_ID = o.USER_ID
```

テーブル名と別名を入力したら、「**別名更新**」ボタンをクリックしてください。
入力した別名（u, oなど）が他のセクションのプルダウンで選択できるようになります。

**取得カラムの指定**
```
テーブル別名: u
カラム名: USER_NAME
別名(AS): ユーザー名
```

**抽出条件の指定**
```
カラム名: STATUS
演算子: =
値: 1
```

#### 3. SQL生成
「SQL生成」ボタンをクリックすると、入力内容に基づいてSQLが生成されます。

### 生成例

#### 基本的なSELECT
```sql
SELECT u.USER_ID,
       u.USER_NAME,
       u.EMAIL
FROM USERS u
WHERE u.STATUS = 1
ORDER BY u.USER_ID ASC;
```

#### JOINと集計を含むSELECT
```sql
SELECT u.USER_ID,
       u.USER_NAME,
       COUNT(*) AS 注文件数,
       SUM(o.AMOUNT) AS 合計金額
FROM USERS u
INNER JOIN ORDERS o ON u.USER_ID = o.USER_ID
WHERE u.STATUS = 1
  AND o.ORDER_DATE >= SYSDATE - 30
GROUP BY u.USER_ID, u.USER_NAME
HAVING SUM(o.AMOUNT) > 10000
ORDER BY SUM(o.AMOUNT) DESC;
```

#### WITH句とサブクエリを含むSELECT
```sql
WITH active_users AS (
    SELECT USER_ID, USER_NAME FROM USERS WHERE STATUS = 1
)
SELECT a.USER_ID,
       a.USER_NAME,
       (
           SELECT COUNT(*) FROM ORDERS WHERE USER_ID = a.USER_ID
       ) AS 注文件数
FROM active_users a
WHERE EXISTS (
    SELECT 1 FROM ORDERS WHERE USER_ID = a.USER_ID
);
```

## SQL構文クイックリファレンス

### JOIN（結合）の種類

| 種別 | 説明 |
|-----|------|
| INNER JOIN | 両方のテーブルに存在するデータのみ取得 |
| LEFT JOIN | 左テーブルの全データ＋右テーブルの一致データ |
| RIGHT JOIN | 右テーブルの全データ＋左テーブルの一致データ |
| FULL OUTER JOIN | 両方のテーブルの全データ |
| CROSS JOIN | 全ての組み合わせ（直積） |

### WHERE句の演算子

| 演算子 | 説明 | 例 |
|-------|------|-----|
| = | 等しい | STATUS = 1 |
| <> | 等しくない | STATUS <> 0 |
| >, <, >=, <= | 大小比較 | AMOUNT > 1000 |
| LIKE | パターン一致 | NAME LIKE '%田中%' |
| IN | リスト内に存在 | STATUS IN (1, 2, 3) |
| BETWEEN | 範囲内 | AGE BETWEEN 20 AND 30 |
| IS NULL | NULLかどうか | DELETE_DATE IS NULL |
| EXISTS | サブクエリに結果が存在 | EXISTS (SELECT 1 FROM ...) |

### 集計関数

| 関数 | 説明 |
|-----|------|
| COUNT(*) | 行数をカウント |
| COUNT(カラム) | NULL以外の件数 |
| COUNT(DISTINCT カラム) | 重複を除いた件数 |
| SUM(カラム) | 合計値 |
| AVG(カラム) | 平均値 |
| MAX(カラム) | 最大値 |
| MIN(カラム) | 最小値 |

## 注意事項

- 生成されたSQLは参考用です。実行前に必ず内容を確認してください
- 件数制限でROWNUM方式を選択した場合、WHERE句に手動で追加が必要です
- サブクエリを使用する場合は、先に「サブクエリ」シートに定義を登録してください
- WITH句やUNIONを使用する場合は、メインシートのオプションで「使用する」を選択してください

### テーブル選択時のカラム自動絞り込み機能

テーブルを選択すると、カラムドロップダウンが自動的にそのテーブルのカラムのみに絞り込まれます。

**この機能を使用するには、VBAプロジェクトへのアクセス許可が必要です：**

1. 「ファイル」→「オプション」→「トラストセンター」→「トラストセンターの設定」
2. 「マクロの設定」→「**VBAプロジェクト オブジェクト モデルへのアクセスを信頼する**」にチェック
3. Excelを再起動し、`InitializeSQL生成ツール`を再実行

**アクセスが許可されていない環境では：**
- テーブル選択後、「**カラム絞込**」ボタンをクリックして手動でカラムを更新できます

## トラブルシューティング

### マクロが実行できない
- Excelのセキュリティ設定でマクロが無効になっている可能性があります
- 「ファイル」→「オプション」→「セキュリティセンター」→「マクロの設定」で確認してください

### シートが作成されない
- `InitializeSQL生成ツール`マクロを実行してください
- エラーが発生する場合は、他のシートを閉じて再度実行してください

### 生成されたSQLにエラーがある
- 入力内容を確認してください
- 特にJOIN条件やGROUP BY句の指定が正しいか確認してください

### モジュール構成

本ツールは2つのVBAモジュールで構成されています：

| モジュール | 役割 | 主な機能 |
|---------|------|---------|
| **SQL_生成ツール_Setup.bas** | 初期化・UI設定 | シート作成、フォーマット設定、ドロップダウン追加、ボタン追加 |
| **SQL_生成ツール.bas** | ビジネスロジック | SQL生成、テーブル定義連動、インポート機能、履歴管理 |

**定数**: 両モジュールで使用する定数（シート名、行番号など）は`SQL_生成ツール_Setup.bas`で`Public`として定義されており、`SQL_生成ツール.bas`から参照できます。

## ライセンス

このツールは自由に使用・改変できます。
