# Git 空フォルダ管理ツール

空のフォルダを検出し、`.gitignore`ファイルを作成することでGitリポジトリで空フォルダを管理できるようにするツールです。

## 概要

Gitは空のフォルダを追跡しません。このツールは、リポジトリ内の空フォルダを検出し、`.gitignore`ファイルを自動作成することで、フォルダ構造をGitで管理できるようにします。

## 必要な環境

- Windows 10/11
- PowerShell 5.1以降（Windows標準搭載）

## 使い方

### 基本的な使用方法

1. `Git_空フォルダ管理ツール.bat` をダブルクリックで実行
2. 空フォルダの一覧が表示される
3. `y` を入力して `.gitignore` を作成

### 対象フォルダの指定

デフォルトではカレントディレクトリが対象です。別のフォルダを対象にする場合は、バッチファイル内の設定を編集してください。

```powershell
# 対象フォルダ（空の場合はカレントディレクトリ）
$targetPath = "C:\path\to\your\repository"
```


## 除外されるフォルダ

以下のフォルダは自動的に除外されます：

| パターン | 説明 |
|---------|------|
| `.git` | Gitリポジトリフォルダ |
| `node_modules` | Node.jsパッケージ |
| `.vs` | Visual Studio設定 |
| `.vscode` | VS Code設定 |
| `__pycache__` | Pythonキャッシュ |
| `.idea` | JetBrains IDE設定 |

### 除外パターンのカスタマイズ

バッチファイル内の設定セクションで除外パターンを追加・変更できます：

```powershell
$excludePatterns = @(
    "^\.git$",
    "^node_modules$",
    "^\.vs$",
    # 追加の除外パターン
    "^bin$",
    "^obj$"
)
```

## 注意事項

- 既に `.gitignore` が存在するフォルダはスキップされます
- `.git` フォルダ配下は自動的に除外されます
- 処理前に確認プロンプトが表示されます
